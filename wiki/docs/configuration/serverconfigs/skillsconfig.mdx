# Skills Config Syntax and Examples

The skills config holds two related maps:
- **`skills`** — every skill the player can earn XP in.
- **`types`** — optional groupings of skills (e.g. "Warfare", "Harvesting") used to organize the inventory skill panel under named, ordered headers.

Both live in the same file (`skills.json`). If `types` is omitted, every skill renders untyped at the bottom of the inventory panel.

## Datapack Example File
```json
{
    "type": "SKILLS",
    "skills": {
        "my_custom_skill": {
          "maxLevel": 2000,
          "icon": "minecraft:textures/particle/enchanted_hit.png",
          "iconSize": 8,
          "useTotalLevels": false,
          "color": 255,
          "noAfkPenalty": false,
          "displayGroupName": false,
          "showInList": true
        },
        "my_custom_skill_group": {
          "maxLevel": 3000,
          "icon": "pmmo:textures/skills/missing_icon.png",
          "iconSize": 18,
          "useTotalLevels": false,
          "groupFor": {
            "archery": 0.2,
            "endurance": 0.3,
            "combat": 0.5
          },
          "color": 16777215,
          "noAfkPenalty": false,
          "displayGroupName": false,
          "showInList": false
        }
    },
    "types": {
        "warfare": {
            "order": 0,
            "color": 13382451,
            "skills": ["combat", "slayer", "hunter", "archery", "gunslinging"]
        },
        "harvesting": {
            "order": 2,
            "color": 13407027,
            "skills": ["mining", "woodcutting", "excavation", "farming", "fishing"]
        }
    }
}
```

## Scripting Example File
```
WITH config(skills)
    set(mining)
        .color(123456)
        .maxLevel(100)
        .noAfkPenalty(true);
    set(fightgroup)
        .groupFor(combat,0.5,archery,0.5)
        .useTotalLevels(true)
        .showInList(false);

    set(warfare)
        .skillType()
        .order(0)
        .color(13382451)
        .skills(combat,slayer,hunter,archery,gunslinging);
END
```

## Skill Scripting Syntax
Each skill setting starts with `set(skillname)` followed by an optional number of value nodes. The table below lists the value nodes, the format for their values and the default value if you omit that keyword.

|    node keyword    | value format                                                                              | default value                                | What it does                                                                         |
|:------------------:|:------------------------------------------------------------------------------------------|:---------------------------------------------|:-------------------------------------------------------------------------------------|
|      `color`       | whole number                                                                              | `16777215`                                   | Sets the skill's display color                                                       |
|     `iconSize`     | whole number                                                                              | `18`                                         | Lets pmmo know the size of the raw image scaling                                     |
|     `maxLevel`     | whole number                                                                              | same as server config max level              | The max level for this skill independent of the global max level                     |
|   `noAfkPenalty`   | true/false                                                                                | `false`                                      | If true this skill will never be affected by anti-cheese                             |
| `displayGroupName` | true/false                                                                                | `false`                                      | Should the group name show on tooltips, if false each member skill will show instead |
|  `useTotalLevels`  | true/false                                                                                | `false`                                      | Should this skill be a sum of its group members when checking skill levels           |
|    `showInList`    | true/false                                                                                | `true`                                       | Should this skill appear in the on-screen HUD list and the inventory skill panel     |
|       `icon`       | namespace:textures/texurename.png                                                         | `minecraft:textures/skills/missing_icon.png` | the location of your desired texture from vanilla or a resource pack                 |
|     `groupFor`     | skills and ratios separated by commas. eg `groupFor(combat,0.3,archery,0.3,endurance,0.4)` | none.                                        | If populated, defines this skill as a "skill group"                                  |

## Skill Types

A skill type is a named grouping that controls how skills are displayed in the inventory panel. Types render as headers, with their listed skills grouped beneath them in the declared order. Skills that aren't listed under any type fall into a trailing untyped section sorted alphabetically.

The display name for a type comes from a translation key — `pmmo.type.<typeKey>` — so localization is handled by your resource pack's `lang` files. There is no `displayName` field on the data itself.

### Type Datapack Fields

|  field   | value format                                            | default value     | What it does                                                                                                                  |
|:--------:|:--------------------------------------------------------|:------------------|:------------------------------------------------------------------------------------------------------------------------------|
| `order`  | whole number                                            | `Integer.MAX_VALUE` | Sort key. Lower numbers render first. Ties are broken by type key alphabetically.                                              |
| `color`  | whole number                                            | `16777215`        | The accent color used for the type's header bar and the colored frame around its skills in the inventory panel.                |
| `skills` | array of skill keys                                     | `[]`              | The skills that belong to this type, rendered in the listed order.                                                             |

### Type Scripting Syntax

In `.pmmo` files, types live under the same `WITH config(skills)` block as skills. To distinguish a type entry from a skill entry, append `.skillType()` to the chain. The `.skillType()` keyword takes no value — its presence is the trigger.

```
WITH config(skills)
    set(harvesting)
        .skillType()
        .order(2)
        .color(13407027)
        .skills(mining,woodcutting,excavation,farming,fishing);
END
```

|  node keyword | value format                                | default value       | What it does                                                                  |
|:-------------:|:--------------------------------------------|:--------------------|:------------------------------------------------------------------------------|
| `skillType`   | (no value)                                  | n/a                 | Marks this `set(...)` block as a type definition rather than a skill.         |
|   `order`     | whole number                                | `Integer.MAX_VALUE` | Sort order among types in the inventory panel.                                |
|   `color`     | whole number                                | `16777215`          | Accent color for the type's header and skill frames.                          |
|   `skills`    | skill keys separated by commas              | none                | The skills that belong to this type, rendered in the order listed.            |

If you forget `.skillType()`, your block will be silently registered as a skill named after the type key instead.

## Hiding Skills

Set `showInList` to `false` on a skill to hide it from both the on-screen HUD list and the inventory skill panel. This is the right setting for `groupFor` parent skills like `fightgroup` that exist only as XP routers, not as skills the player should see in the UI.
